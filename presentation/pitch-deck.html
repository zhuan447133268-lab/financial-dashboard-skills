规则预警

"我故意在示例数据里埋了两个坑：表头有空格、预算成本里有'暂估'，看 Skill 能不能识别出来。"

Step 1：让 AI 识别数据

python skills/financial-data-teller/data_teller.py \
  --input examples/company-dashboard/input \
  --output examples/company-dashboard/output/数据资产说明书.md
  • 自动识别表类型：预算表、实际发生表、组织基础信息表
  • 自动标注字段语义：时间、主体、金额、科目
  • 自动输出异常清单和可分析维度候选

它发现了什么？

⚠️

高风险异常

表头存在首尾空格，可能导致按列名读取失败

📝

中风险异常

金额字段存在非数字占位值 "暂估"，影响汇总计算

"这个 Skill 不是默默帮你清洗，而是先把问题暴露出来，让关键口径由人来确认。"

Step 2：生成可交互看板

python skills/financial-dashboard-builder/dashboard_builder.py \
  --config examples/company-dashboard/output/看板配置.json \
  --output examples/company-dashboard/output/经营分析看板.html
  • 单文件 HTML，内嵌 ECharts，浏览器直接打开
  • 4 个 KPI 卡片 + 4 个交互图表
  • 规则驱动的核心发现与建议

看板长什么样？

💰

KPI 卡片

实际收入、预算收入、收入达成率、成本偏差率

📈

趋势图表

月度收入/成本趋势、预算达成率、城市对比、主体排行

🚨

自动预警

"实际成本超出预算 22.6%，建议核查成本科目"

📱

响应式布局

大屏投影、笔记本、移动端都能正常展示

为什么这个 Skill 好用？

🪶

轻量，无依赖

纯 Python，只需 pandas + openpyxl + jinja2，无需 MCP server / Node.js，拿到就能跑

✓

可验证，敢用

只做事实识别和规则判断，所有数字带来源、公式、异常报告，不接 AI 自由生成

🎯

一个 Skill 一个岗位

识数据的、做看板的职责分离，可独立复用，也可串联成工作流

它改变了什么？

环节 传统方式 用 Skill 之后
理解数据 人工逐张看表 AI 自动生成数据资产说明书
口径确认 反复沟通 异常清单 + 候选维度，一次确认
生成看板 手动画图 一条命令生成可交互 HTML
经营建议 凭经验写 规则引擎自动触发预警

开源与下一步
