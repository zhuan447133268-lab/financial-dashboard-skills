# 数据质量检查清单

financial-data-teller 在读取 Excel 时必须检查以下数据质量问题。

## 致命问题（必须拦截并报告）

- [ ] **表头空格**：列名存在首尾空格，导致按列名读取失败
- [ ] **合并单元格**：表头或数据区域存在合并单元格，导致数据错位
- [ ] **空列/空行**：大量空列或空行影响表头检测
- [ ] **重复列名**：同一张表出现重复列名

## 高风险问题（必须报告）

- [ ] **单位不一致**：同一指标在不同表中单位不同（元 vs 万元）
- [ ] **金额非数字**：金额字段存在 "-"、""、"暂估"、"None" 等占位值
- [ ] **时间断档**：缺少关键月份或季度
- [ ] **主键缺失**：需要关联的表缺少主体编码、月份等关联字段

## 中风险问题（建议报告）

- [ ] **空值率高**：某列空值率超过 30%
- [ ] **异常极值**：金额出现明显异常的极大/极小值
- [ ] **表头多层**：表头跨多行，需要确认实际表头行
- [ ] **字段名不规范**：使用缩写、拼音、特殊符号

## 处理原则

1. **不自动清洗**：只报告异常，不默认填充或删除
2. **置信度标注**：所有推断都要标注置信度
3. **停下来确认**：高风险问题必须让用户确认后再继续
